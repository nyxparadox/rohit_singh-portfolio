Smart Doorbell System

IoT-based Visitor Monitoring & Notification System

Problem

Traditional doorbells provide no context or history of visitors. This project addresses that limitation by enabling real-time visitor monitoring and cloud-based logging through a mobile application.

What I Built

I designed and implemented an end-to-end IoT system that captures visitor images using an ESP32-CAM, securely stores them in the cloud, logs events in a database, and delivers real-time notifications to a Flutter-based mobile application.

System Workflow

  • Visitor presses the doorbell
  • ESP32-CAM captures an image
  • Image is uploaded to Cloudinary
  • Event is logged in Firebase Firestore
  • Python service triggers FCM notifications
  • Flutter app displays visitor details

Key Features

  • Real-time push notifications
  • Cloud-backed visitor history
  • Cross-platform Flutter mobile app
  • Secure authentication and encrypted communication

Technology Stack

ESP32-CAM • Arduino • Firebase Firestore • Firebase Cloud Messaging • Cloudinary • Python • Flutter

What This Project Demonstrates

This project demonstrates my ability to design complete IoT systems, integrate embedded hardware with cloud services, and build secure, real-time mobile applications.

MelodySync (In Progress)

Voice-Activated Music Synchronization System

Problem

Existing music recognition systems can identify songs but cannot synchronize playback with a user’s singing. MelodySync explores a new way of interacting with music through vocal input.

What I’m Building

MelodySync is an AI-based system that listens to a short segment of a user singing, identifies the song, determines the exact timestamp in the original track, and resumes playback from that precise position.

How It Works

  • Wake word activates the system
  • Short vocal sample is recorded
  • Pitch contours and melodic patterns are extracted
  • Patterns are matched against a song database
  • Playback resumes from the detected timestamp

What Makes It Different

Unlike traditional systems that rely on lyrics or audio fingerprinting, MelodySync focuses on melodic structure, allowing recognition from humming or singing alone.

Current Status

The current implementation runs in a terminal environment, prioritizing accuracy, synchronization precision, and low latency. UI and streaming integrations are planned for future versions.

Technical Domains

Python • Digital Signal Processing • Music Information Retrieval • Time-Series Analysis

What This Project Demonstrates

MelodySync demonstrates my ability to work with complex signal-processing concepts, design algorithm-driven systems, and explore innovative human–computer interaction models.
